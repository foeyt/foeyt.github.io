我用 Kimi 编了一个口诀，便于记忆优先级和结合性：

  • 优先级：括号老大最优先，单目老二站一边， 乘除老三跑得快，加减老四跟后面， 移位老五左右晃，关系老六比高矮， 相等老七辨真假，位运老八逐位算， 逻辑老九十关联，条件十一选三元， 赋值十二排最后，逗号老幺站队尾。
  • 结合性：左结合是主力军，单目条件赋值从右向左行。

由这些优先级、结合性等引导出下面的问题：

  • 数组指针与指针数组：*p[] 是指针数组, (*p)[] 是数组指针。主体分别为数组和指针，指针数组本质上是数组，内部元素为指针，数组指针本质为指针，指向一个数组。另外，对一个数组取地址运算得到的数据类型是数组指针而非指向指针的指针（这个之前实际 coding 过程碰到过）。
  • 利用指针遍历数组，可以采用 *p++ 的方式，这样 p 会逐个指向数组中的元素（类似于 Python 中的 Iterator？）。
  • 逻辑运算符的短路特性：见例题：m, n, a, b, c, d 整形变量起始值都为0，那么 (m = a == b) || (n = c == d) 的执行结果是 m = 1, n = 0，首先左结合计算 m = a == b，右结合计算 a == b，返回值为 true（在 C 中为 1），m = 1，触发逻辑或运算短路，右边不参与计算。
  • 逻辑运算符的操作数可以是任何标量类型（可比较类型），但是返回值一定是整型（其实是布尔值），所有非零的操作数都将作为 true，所有为零的操作数都将作为 false。

格式字符与格式标识符

  • 格式字符：

    格式字符 意义
    a, A 以十六进制形式输出浮点数(C99 新增)。实例 printf(“pi=%a”, 3.14); 输出 pi=0x1.91eb86p+1。
    d 以十进制形式输出带符号整数(正数不输出符号)
    o 以八进制形式输出无符号整数(不输出前缀0)
    x,X 以十六进制形式输出无符号整数(不输出前缀Ox)
    u 以十进制形式输出无符号整数
    f 以小数形式输出单、双精度实数
    e,E 以指数形式输出单、双精度实数
    g,G 以%f或%e中较短的输出宽度输出单、双精度实数
    c 输出单个字符
    s 输出字符串
    p 输出指针地址
    lu 32位无符号整数
    llu 64位无符号整数

  • 格式标识符：

    flags（标识） 描述
    - 在给定的字段宽度内左对齐，默认是右对齐（参见 width 子说明符）。
    + 强制在结果之前显示加号或减号（+ 或 -），即正数前面会显示 + 号。默认情况下，只有负数前面会显示一个 - 号。
    空格 如果没有写入任何符号，则在该值前面插入一个空格。
    # 与 o、x 或 X 说明符一起使用时，非零值前面会分别显示 0、0x 或 0X。 与 e、E 和 f 一起使用时，会强制输出包含一个小数点，即使后边没有数字时也会显示小数点。默认情况下，如果后边没有数字时候，不会显示显示小数点。 与 g 或 G 一起使用时，结果与使用 e 或 E 时相同，但是尾部的零不会被移除。
    0 在指定填充 padding 的数字左边放置零（0），而不是空格（参见 width 子说明符）。

  • 输出宽度：

    width（宽度） 描述
    (number) 要输出的字符的最小数目。如果输出的值短于该数，结果会用空格填充。如果输出的值长于该数，结果不会被截断。
    * 宽度在 format 字符串中未指定，但是会作为附加整数值参数放置于要被格式化的参数之前。

  • 输出精度：

    .precision（精度） 描述
    .number 对于整数说明符（d、i、o、u、x、X）：precision 指定了要写入的数字的最小位数。如果写入的值短于该数，结果会用前导零来填充。如果写入的值长于该数，结果不会被截断。精度为 0 意味着不写入任何字符。 对于 e、E 和 f 说明符：要在小数点后输出的小数位数。 对于 g 和 G 说明符：要输出的最大有效位数。 对于 s: 要输出的最大字符数。默认情况下，所有字符都会被输出，直到遇到末尾的空字符。 对于 c 类型：没有任何影响。 当未指定任何精度时，默认为 1。如果指定时不带有一个显式值，则假定为 0。
    .* 精度在 format 字符串中未指定，但是会作为附加整数值参数放置于要被格式化的参数之前。

  • 函数的定义与声明

    函数声明会告诉编译器函数名称及如何调用函数。函数的实际主体可以单独定义。若函数直接被定义，那么会自动声明，注意执行是有先后的，前面的函数不能直接调用后面的函数，除非这个函数已经被声明。

    在 C 语言中，函数由一个函数头和一个函数主体组成。下面列出一个函数的所有组成部分：

    • 返回类型：一个函数可以返回一个值。return_type 是函数返回的值的数据类型。有些函数执行所需的操作而不返回值，在这种情况下，return_type 是关键字 void。
    • 函数名称：这是函数的实际名称。函数名和参数列表一起构成了函数签名。
    • 参数：参数就像是占位符。当函数被调用时，您向参数传递一个值，这个值被称为实际参数。参数列表包括函数参数的类型、顺序、数量。参数是可选的，也就是说，函数可能不包含参数。
    • 函数主体：函数主体包含一组定义函数执行任务的语句。

  • 全局变量与静态变量：

    C 语言中是没有 global 关键字的，所有的全局变量都位于函数体之外，作用域为整个程序。所有的全局变量都是静态的。

    静态变量可以当作 Java 的静态方法区来理解，static 变量会划分一个专门的区域来存储，作用域限制在对应语句块内，但是不会在作用域结束后被自动释放，直到程序停止运行后自动释放。另外静态变量默认起始值为 0。并且重复定义静态变量是不会改变值的，比如我重复运行语句 static int x = 1，只有在第一次运行时 x 为 1，此后执行 x++ 后，再次运行这个定义语句，x 不会变成 1，而是为 2。

    此外关于变量的作用域，和作用域结束后自动释放变量，我觉得利用 Rust 的生命周期概念解释比较好。把作用域看作是变量的生命周期，作用域结束即生命周期结束，自动释放变量，此后再次调用变量会出现未定义。

  • 自动类型转换：

    自动类型转换是由编译器自动完成的，无需程序员显式指定，这种转换通常发生在不同类型的数据进行混合运算时。C语言中的自动类型转换遵循一定的规则，主要是为了保证数据的精度和范围。

    • 当一个表达式中包含不同的数据类型时，较小的类型会自动转换为较大的类型。
    • 所有的浮点运算都是以双精度进行的，即使运算中只有 float 类型。
    • 在赋值操作中，右边表达式的类型将自动转换为左边变量的类型。

算法

  • 时间复杂度与空间复杂度，表示算法运行的时间规模与空间规模的度量，例如冒泡排序的时间复杂度是 O(n2)，空间复杂度为 O(1)，这是因为冒泡排序最坏情况需要两层循环，但是只需要一个变量来实现交换数据。
    • 举例，二分查找算法的时间复杂度为 O(log2n)，这意味着一个 100 个元素的数组进行二分查找，最多执行 ⌈log2(100)⌉ = 7 次。

预处理

预处理命令都从 # 开始，但是预处理和语句一样可以通过  续行。并且预处理命令不一定就要位于所有语句之前，比如 #endif。预处理本身不是 C 语言的组成部分，它是 C 语言运行过程的环境架设，就相当于 Java 的 JVM，Python 的 venv 一样。
