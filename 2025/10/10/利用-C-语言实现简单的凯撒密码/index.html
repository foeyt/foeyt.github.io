利用 C 语言实现简单的凯撒密码

布置的一道扩展题，用于实现凯撒加密。

凯撒密码就是一个简单的替换加密，这里题设 key: 'A' = 'D' 所以不需要再搞 key 这一块。我的算法比网上某些转换器精准一点，可以无损转所有的 ASCII 字符，同时跳过所有非 ASCII 字符。

最重头戏的是我搞了一个可变长度的 *buffer 用于存放字符串，这是我第一次尝试搞内存上的编程，先抄了网上的作业。

惯例，先上代码！

可变长度字符串

这个函数用于获取输入，同时根据输入长度改变字符串长度。

我需要慢慢解释一下：

先定义了一个静态的全局字符型指针 *buffer，先让这个指针悬空，把这个指针存栈里面。定义全局是为了在 main() 中最后释放内存。

定义了 3 个变量 size，len，temp，其中 size 和 len 为 size_t 类型，这和内存分配有关，其中 size 负责字符串容量，len 负责字符串的长度，当长度太长时主动扩大 size ，分配更多内存。char temp 这个字符型变量负责逐个读取输入，并将输入塞进 *buffer 指向的堆里面。

随后定义一个 while 死循环，不断进行操作，在最后选择 break 跳出，一般经常用这种方式写一些持续时间长的代码，比如我曾经在 Java 里用这种方式搞了一个 Socket 服务器，也是利用 while 死循环，catch 到异常时跳出。

这串是负责分配内存的主要代码，第一个 if 用于判断是否需要扩容，即字符串的长度比字符串本身还大或两个一样大了的情况有没有发生。如果有，进行扩容。

定义了一个 n_size，就是我们下面要扩容的容量，这里说一下 realloc() 函数。

将 size 刷新为新容量。

这里用到了一个 scanf() 的特性，翻一下 <stdio.h> 发现：

返回值居然是一个 int 类型，也就是返回值说明了这个函数是否执行成功，也就是我们用于这里将其放在 if 中的原因，同时判断输入合法情况与把读取一个输入的字符给放到一起了，挺好。

同时 scanf("%c", &temp) 还有一个特性，我们输入的内容都将进入缓冲区，函数每次会从缓冲区中读取任何一个字符，包括空格和换行，这时 %c 独有的。比如我输入了 Razor sax，把这个 scanf("%c", &temp) 放到一个循环里面，首先 Razor sax 会进入缓冲区，随后函数读取缓冲区的一个字符 'R'，此时 temp = 'R'，并将 'R' 字符从缓冲区中移除，循环再次运行到这里时，缓冲区只剩下了 azor sax，同理继续读取。

buffer[len++] = temp 也是一个很好的写法，同时实现了两个功能，即 buffer[len] = temp，和 len++，这样将每次读取的字符存到 *buffer 指向的堆区域，这样 *buffer != NULL 了，字符串也就存进去了。

后面的 if 判断是否输入了回车，回车就结束输入，并且按照 C 语言的特性将字符串末尾换成 '\0'，方便进行判断是否遍历到头。

最后不要忘了在 main() 里面把 *buffer 指向的内存给释放了。

替换加密

这部分就比较简单了，就是许多条件判断而已。

首先遍历字符串，这里我们把末尾换成 '\0' 的好处就出现了，遍历很方便。首先判断是不是 ASCII 中的字母，不是就不要管，然后判断是不是三个特殊的字母 x, y, z，因为 A = D，所以 x, y, z = a, b, c，如果有一个环形的数据结构就好了~就不用选择这些特殊字符了，或许可以用环形链表，这样就好搞了。之后试着优化一下代码。
